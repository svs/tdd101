Rspec Basics

describe
context
before and after hooks 
describe Class do

  before(:all) { @post = FactoryGirl.create(:post) }
  after(:all) { @post.destroy }

  context "when/with something" do

    before(:each) { @post.something }

    specify { @post.should be_valid }
    it { should_not be_valid }

How does this work?

describe creates a subclass of RSpec::Core::ExampleGroup and executes the block in it's context.
context creates a subgroup which is a subclass of the outer class.
it and specify blocks execute in the context of an instance of the group in which they are declared.
